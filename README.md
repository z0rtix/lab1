# Лабораторная работа №1: Полиморфный динамический массив (C) (11 вариант)

Реализация универсального динамического массива на языке C с поддержкой различных типов данных через механизм полиморфизма (пользовательские `TypeInfo`, `void*` и указатели на функции).  
Выполнено в рамках курса «Информатика (абстрактные типы данных)» (2 семестр) в соответствии с вариантом №11.

## Структура проекта

- `DynamicArray/DynamicArray.h` – заголовок структуры `DynamicArray` и прототипов базовых операций.
- `DynamicArray/DynamicArrayCore.c` – реализация управления памятью: `create`, `destroy`, `push`, `pop`, `remove_at`, `get`, `set`, `clear`, `reserve`, `shrink_to_fit`.
- `DynamicArray/DynamicArrayAlgorithms.c` – алгоритмы: `sort`, `copy`, `map`, `where`, `concat`.
- `TypeInfo/TypeInfo.h` – описание структуры `TypeInfo` (размер элемента, функции сравнения, печати, копирования, освобождения) и геттеры для ленивой инициализации типов.
- `TypeInfo/TypeInfo.c` – реализация геттеров для `int`, `double`, `string` и массива массивов.
- `utils/utils.h` – функции для конкретных типов (`int_compare`, `int_print`, `int_copy`, `string_map_upper`, предикаты и т.п.).
- `utils/utils.c` – реализации всех вспомогательных функций.
- `UI_menu/menu.h` / `menu.c` – консольное меню с выбором типа, добавлением, удалением, сортировкой, `map`, `where`.
- `Tests/tests_of_int.c` / `tests_of_str.c` – модульные тесты для `int` и `string` (покрытие всех операций, граничные случаи, стресс-тест 10 млн операций).
- `main.c` – точка входа (запуск тестов и интерактивного меню).
- `Makefile` – сборка проекта (флаги `-std=c11 -Wall -Wextra -g -O2 -D_GNU_SOURCE`).

1. **Модульные тесты** (`Tests/tests_of_int.c`, `Tests/tests_of_str.c`) успешно проходят, *утечек памяти нет* (проверено Valgrind и AddressSanitizer).
2. **Базовые операции**: `create`, `destroy`, `push`, `pop`, `remove_at`, `get`, `set`, `size`, `empty`, `capacity`, `clear`, `reserve`, `shrink_to_fit`.
3. **Алгоритмы**: `sort`, `copy`, `map`, `where`, `concat`.
4. **Типы данных**: `int`, `double`, `string`, *массивы массивов*.
5. **Граничные случаи**: пустой массив, выход за пределы индекса, некорректный ввод в меню.
6. **Производительность**: 10 млн операций push для int – <0.3 сек, для строк – ~2.8 сек (разумно из-за strdup/free).


## Консольное меню
### Позволяет:
- Выбрать тип данных (`int`, `double`, `string`).
- Создать *пустой* массив или работать с текущим.
- *Добавить* элемент, *удалить* последний, *удалить* по индексу, *изменить* элемент.
- *Сортировать* массив.
- *Применить* `map` (возведение в квадрат, куб, прибавление 10; для строк – toUPPER, добавить !!!, реверс).
- *Применить* `where` (*фильтр*: чётные/нечётные, длина >3, начинается с A/a, содержит 'e').
- *Заменить* текущий массив на результат `map` или `where` (с подтверждением).
- В любой момент *сменить* тип массива (создать новый).

## Особенности реализации
1. Полиморфизм через `TypeInfo` и `void*`
2. Все операции с элементами (`compare`, `print`, `free`, `copy`) вынесены в структуру `TypeInfo`, которая создаётся лениво (один экземпляр на тип). Это позволяет добавлять новые типы без изменения кода динамического массива.
3. Разделение владения данными (`is_view`)
4. Массивы, созданные через create или copy, являются владельцами (`is_view = 0`) – при destroy освобождают элементы.
5. Массивы, возвращаемые `where`, являются наблюдателями (`is_view = 1`) – не освобождают элементы и не могут быть изменены (проверка в `push`, `set`, `remove_at`).
6. При *замене* текущего массива на результат `where` в меню *автоматически* создаётся глубокая копия (через `copy`), чтобы сохранить *независимость*.
7. **Ленивая инициализация** `TypeInfo`
8. Глобальные описатели типов (`int_type`, `string_type`) объявлены `static` в `TypeInfo.c`, а доступ к ним осуществляется через функции-геттеры (`get_int_type()` и т.д.). Это гарантирует единственность экземпляра и соответствие методическим указаниям.
9. Константная корректность. Все функции, которые не изменяют массив (например, `get_size`, `print_array`, `get`), принимают `const DynamicArray*`. Функции сравнения и печати в `TypeInfo` принимают `const void*`.

## Обработка ошибок
- Проверка NULL после malloc/realloc и в начале каждой публичной функции.
- Проверка границ индекса в get, set, remove_at.
- В консольном меню – проверка возврата scanf/fgets и очистка буфера ввода.

## Алгоритмы
- `sort` – пузырьковая сортировка с использованием element_compare.
- `map` – создаёт копию массива (через copy) и применяет функцию к каждому элементу.
- `where` – возвращает массив-наблюдатель (без копирования данных).
- `concat` – возвращает новый массив-владелец (глубокое копирование элементов обоих исходных массивов).

## Тестирование
Модульные тесты для каждого типа (int и string) покрывают все операции, граничные случаи и проверяют отсутствие утечек.
Стресс-тест на 10 миллионов push подтверждает эффективность геометрического роста.

## Примечания
Для изменения размера тестовой выборки (10 млн) отредактируйте `Tests/tests_of_int.c` и `Tests/tests_of_str.c`.
Для просмотра результатов работы **запустите** `./main` и выберите интерактивный режим.
При необходимости добавить **новый** тип данных – создайте соответствующие функции *(_compare, _print, _copy, _free)* и **добавьте** геттер в `TypeInfo.c`.
